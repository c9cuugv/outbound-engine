# AI-Assisted Campaign Creation — Design Spec

## Problem

The current campaign wizard Step 1 forces users to write 3 lengthy text fields manually (product description ≥50 chars, ICP, value prop). Step 3 requires manually selecting email templates. This creates friction and doesn't align with the product's promise of AI-powered automation.

## Solution

Replace Steps 1 and 3 of the campaign wizard with a conversational AI chat interface. The AI asks ~4 short questions, generates all campaign fields and email templates automatically, creates the campaign in the background, and advances the user directly to Select Leads.

## User Flow

1. User navigates to `/campaigns/new`
2. Sees an **intro card**: AI introduces itself and explains it will take ~2 minutes with 4 questions. User clicks "Let's go →"
3. **Chat interface** opens. AI asks questions one at a time (streamed via SSE):
   - "What's your campaign called, and what are you selling?"
   - "Who's your ideal buyer — role, company size, industry?"
   - "What's the main reason they should care? What problem do you solve?"
   - "How many emails in the sequence and what tone? (default: 3, professional-casual)"
4. After the 4th answer: AI says "Got it! Creating your campaign now..." and **silently creates the campaign + triggers email generation** in the background
5. Frontend **auto-advances to Select Leads** with the new campaign_id in state
6. Steps 2 (Select Leads) and 4 (Sending Settings) remain unchanged

## What Changes

### Wizard structure: 4 steps → AI Chat + 2 steps
- BEFORE: Product Info → Select Leads → Sequence → Sending Settings
- AFTER: AI Chat (pre-step, creates campaign) → Select Leads → Sending Settings

### New wizard steps numbering:
- Step 0 (pre): AI Chat — creates campaign + generates templates in background
- Step 1: Select Leads (unchanged)
- Step 2: Sending Settings (unchanged)
- Final: Launch (unchanged)

## Backend Design

### New endpoint: `POST /api/v1/campaigns/chat`

**Request:**
```json
{
  "messages": [
    {"role": "user", "content": "Q1 SaaS Push — OutboundEngine..."},
    {"role": "assistant", "content": "Great! Who is your ideal buyer?"},
    {"role": "user", "content": "Series A SaaS founders..."}
  ]
}
```

**Response:** `text/event-stream` (SSE)

**SSE event types:**
```
data: {"type": "token", "content": "Who is your ideal"}
data: {"type": "token", "content": " buyer?"}
data: {"type": "done", "campaign_id": "uuid-here"}
```

**Server logic:**
1. Receive conversation history
2. Determine which question to ask next (based on message count / content)
3. If < 4 answers collected: stream next question via LLM
4. If 4 answers collected: extract structured data, create campaign via existing `create_campaign` service, trigger `generate_campaign_emails`, stream "done" event with campaign_id

**LLM System Prompt** (abbreviated):
> You are helping set up an outbound email campaign. Ask exactly 4 questions in order. After the 4th answer, output a special JSON block: `<CAMPAIGN_DATA>{"name":...,"product_name":...,"product_description":...,"icp_description":...,"value_prop":...,"sequence_count":3,"tone":"professional-casual"}</CAMPAIGN_DATA>`. Never skip questions. Never ask more than 4.

**Uses existing config:** `EMAIL_GEN_PROVIDER` + `GEMINI_API_KEY` / `GROQ_API_KEY` / `ANTHROPIC_API_KEY`

## Frontend Design

### New files:
- `frontend/src/components/campaign/CampaignChatSetup.tsx` — intro card + chat bubbles + SSE stream consumer
- `frontend/src/hooks/useCampaignChat.ts` — manages SSE connection, message state, extracts done event

### Modified files:
- `frontend/src/pages/CampaignWizard.tsx` — replace steps 1 & 3, new pre-step flow

### CampaignChatSetup states:
1. `idle` — shows intro card with "Let's go" button
2. `chatting` — chat bubbles, streaming AI text, user input box
3. `creating` — AI says "Creating your campaign...", spinner
4. `done` — emits `onComplete(campaignId)` to parent

### CampaignWizard changes:
- Add `campaignId` to state (set after chat completes)
- Step 1 renders `<CampaignChatSetup onComplete={(id) => { setCampaignId(id); setStep(2); }} />`
- Step 3 (template selection) removed — templates auto-generated by backend
- Step numbering: 1=Chat, 2=Select Leads, 3=Sending Settings

## No Changes To:
- EmailReviewQueue — users still review generated emails there
- CampaignDashboard
- Any backend models or DB schema
- Steps 2 and 4 of wizard
